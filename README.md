# My Campus Blog

The app works as an offline blog/social message client where users can create, store, search, edit, delete, attach images to, and share personal posts.

## Coursework Features

- Create text-based blog posts.
- Edit existing posts.
- View a list of saved posts.
- View one individual post in detail.
- Search posts by title or message content.
- Delete a single post.
- Select multiple posts and delete them together.
- Store all posts locally using SQLite on the device.
- Attach an image from the camera or gallery.
- Share an individual post with text and optional image through Android's native share sheet.
- Organise posts by category.
- Filter posts by category.
- Sort posts by newest, oldest, title A-Z, or title Z-A.

## Technology Used

- Flutter
- Dart
- SQLite with `sqflite`
- `image_picker` for camera/gallery image selection
- `share_plus` for Android sharing
- `path` and `path_provider` for local file/database paths

## Project Structure

```text
lib/
  config/
    post_options.dart
  database/
    database_helper.dart
  models/
    post.dart
  screens/
    add_edit_post_screen.dart
    post_detail_screen.dart
    post_list_screen.dart
  main.dart
```

## Main Files

- `lib/main.dart`: App entry point and theme setup.
- `lib/models/post.dart`: Post data model.
- `lib/database/database_helper.dart`: SQLite database creation, migration, insert, update, search, sort, and delete logic.
- `lib/config/post_options.dart`: Category and sort option definitions.
- `lib/screens/post_list_screen.dart`: Home screen, search, filters, sorting, post list, and multi-delete.
- `lib/screens/add_edit_post_screen.dart`: Create/edit post form and image picker.
- `lib/screens/post_detail_screen.dart`: Individual post view, share, edit, and delete.

## How To Run

From the project folder:

```bash
flutter pub get
flutter run
```

Use an Android emulator or a real Android device with USB debugging enabled.
